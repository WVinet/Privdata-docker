5. ESTRUCTURA FUNCIONAL QUE DEBE QUEDAR
==================================================

A. PANEL ADMIN
Módulos permitidos para MVP:

1. Dashboard
2. Usuarios
3. Titulares
4. Consentimientos
5. Solicitudes ARCO
6. Auditoría

Interpretación correcta de cada módulo:

1) Dashboard
- resumen general del sistema
- métricas de titulares, consentimientos, solicitudes y pendientes
- actividad reciente
- solicitudes recientes

2) Usuarios
- gestión de cuentas internas del sistema
- representa cuentas técnicas de acceso
- debe mostrar nombre, correo, rol, estado, fecha creación
- se puede crear usuario interno
- no debe confundirse con titulares

3) Titulares
- registro/listado de personas titulares de datos
- debe mostrar nombre, rut, email, teléfono, cantidad de consentimientos, fecha registro
- semánticamente representa persons del dominio para el caso de titulares
- puede mantener el nombre visual “Titulares”

4) Consentimientos
- gestión formal de consentimientos otorgados
- tabla administrativa
- puede incluir titular, finalidad, categoría, estado, versión, hash, fecha
- debe alinearse con lo que el titular ve en el portal

5) Solicitudes ARCO
- gestión interna de solicitudes de derechos
- listado administrativo
- responsable
- estado
- vencimiento
- tipos coherentes con portal

6) Auditoría
- registro de eventos del sistema
- acciones relevantes internas y del dominio
- búsqueda por acción, usuario o detalle

B. PORTAL TITULAR
Módulos permitidos para MVP:

1. Inicio
2. Consentimientos
3. Derechos ARCO
4. Seguimiento

Interpretación correcta:

1) Inicio
- bienvenida al titular autenticado
- resumen personal
- accesos rápidos
- no debe sentirse como un dashboard admin

2) Consentimientos
- vista amigable de consentimientos otorgados
- agrupados por categoría
- toggle cuando corresponda
- indicar si es necesario, opcional o no revocable
- mostrar desde cuándo aplica

3) Derechos ARCO
- formulario de solicitud
- selección del derecho desde un catálogo unificado
- datos del titular
- alcance
- descripción
- declaración de veracidad
- experiencia simple y enfocada

4) Seguimiento
- estado de solicitudes enviadas
- línea de avance o timeline
- fechas, plazos, hitos y resultado
- descarga de constancia cuando aplique
